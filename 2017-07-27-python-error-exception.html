概念

错误:

  1. 语法错误
  2. 逻辑错误

异常:

  1. 程序遇到逻辑或者算法问题
  2. 内存不够或者IO错误

错误和异常的处理方式

常见错误

NameError

没有定义名字的情况下进行引用 : is not defined

SyntaxError

语法错误 : invalid syntax

IOError

文件打开错误: No such file or directory

ZeroDivisionError

除零错误: integer division or modulo by zero

ValueError

转换错误: invalid literal for

KeyboardInterrupt

Ctrl + c 中断错误

使用 try-except 处理异常

捕获异常,并处理

出现的异常会被保存在 e 变量中,如果 try 代码块出现异常和 Exception 一致,那么就会对其进行处理,不一致会交给解释器进行处理

如果 except 后面不加任何参数的时候,能捕获运行时的所有错误,但是运行前的错误不能捕获

正确的处理异常可以使代码更加健壮

处理多个异常

try-finally语句

finally 并不处理异常,而是为异常处理事件提供清理机制,用来关闭文件或者释放系统资源

无论是否检测到异常,都会执行 finally 语句

try-except-else-finally语句

如果没有捕获异常,执行完try以后执行finally,如果捕获异常,执行except后,执行finally

如果没有捕获异常,执行try,执行else,然后执行finally

如果捕获异常,执行try,执行except,然后执行finally

with-as 语句

代替 try-except-finally语句,context表达式返回是一个对象,car用来保存context返回对象,单个返回值或者元组

with 语句是指是上下文管理

上下文管理协议:包含方法 exter() 和 exit() ,支持该协议的对象要实现这两个方法

上下文管理器:调用管理器 enter 方法进入,若果设置 as var 语句,var变量接收 enter() 方法返回值,然后执行 exit() 方法退出

with应用场景

  1. 文件操作
  2. 进程线程之间互斥对象,例如互斥锁
  3. 支持上下文的其他对象

标准异常和自定义异常

raise语句

用于主动抛出异常,测试代码对错误的处理

raise exception, <args>

raise ValueError, 'This is a ValueError'

exception表示异常类

args描述异常信息的元组

assert语句

assert 语句用于检查表达式是否为真,如果为假,引发AssertionError错误

assert expression, <args>

assert 7==6, "7永远等于6"

expression:表达式

args:用于对表达式的描述
